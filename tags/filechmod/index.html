Changing default behavior of File::chmod

Sat Jul 6, 2013 by Caleb Cushing in  File::chmod, ironman, perl

File::chmod has been around for a long time, and is really stable, and really hasn’t changed since 1999. It is far more user friendly than the chmod()  in core Perl. I recently used it for an interview test. It took me a few times to get right however because it’s default behavior in symchmod() mode is to use the systems umask.
